• Which medical image file formats does Slicer read?

    Slicer reads both raw data (namely, binary files composed of shorts, unsigned chars, and so forth) and DICOM (digital imaging and communications in medicine) formatted files.
    For raw data (regardless of whether the files have headers or not), Slicer expects the images to be numbered as follows: I.001, I.002, I.003...
    If you have Genesis headers, try the automatic header-reading option.
    See the Tutorial for more information.
